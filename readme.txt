【概要】
  IRCクライアントなどのいわゆるチャットソフトのユーザーインターフェースに
近い形で、共有フォルダなどのファイル共有環境さえあればチャットできる
ソフトを目指しています。
  主にイントラネットでの運用をターゲットとしています。


【環境】
  本ソフトウェアは Visual-C# 2008 express にて .NET Framework 3.5 を
ターゲットに開発しております。
  従いまして利用には、.NET Framework 3.5 のインストールが必要です。

【インストール】
  ディレクトリ丸ごと、どこかにおいてください。
  NekoChat.exe を実行すれば動作します。

【アンインストール】
  ディレクトリごと削除ください。
  本ソフトはレジストリに
    \HKEY_CURRENT_USER\Software\Ryuz\NekoChat
  以下のデータを作成します。
  気になる方はレジストリエディタなどで削除ください。

【使い方】
  本ソフトは、チャンネルごとに共有フォルダを１つ必要とします。
  起動後、チャンネルの追加で共有フォルダのパスを指定することで
会話を開始することが出来ます。


【開発背景】
  チーム開発やリモートオフィスでのコミュニケーションの活性化にチャットは
非常に便利です。メールより手間が少なく、且つ、電話と違いすぐに応答できなく
ても良いという利点があります。

  しかしながら、イントラネットにおいてはチャットサーバーの準備が面倒で
あったり、NetMeeting などのサーバーレスのシステムだと、DHCP環境で運用が面倒
などと使いにくい点があります。
  一方で、Windowsの共有フォルダであれば、比較的どこのイントラネットでも
存在するのではないでしょうか。

  しかも、共有フォルダに置くということは、Active Directory などのアクセス
権制御の恩恵にそのまま与れます。また簡単ながらデータの暗号化にも対応して
いますので、宛先間違いや盗聴のリスクのあるメールや電話に比べて、情報
漏えいリスクの低いより安全な通信手段といえるでしょう。


【技術原理】
  行っている動作は、共有サーバーのテキストファイルに多人数で読み書きを
行い、変更を監視してリロードを行っているだけです。
  Windowsのファイルシステムにはディレクトリの変更監視を行うAPIが
備わっていますので、タイマポーリングほどサーバーに負荷を掛けることなく
実施可能です。
  変更監視付きのテキストエディタでほぼ同じことが実現できるのですが、
ユーザーインターフェース的にチャットっぽく使いやすくしています。
  レスポンスの悪いネットワーク環境も考慮し、読み書き部分を別スレッドに
していますので、使用感は一般的なチャットに近づけられるよにしています。
